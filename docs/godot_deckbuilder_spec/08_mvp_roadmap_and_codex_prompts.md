# 08. MVP 로드맵과 Codex 프롬프트

## 1. 개발 목표

Godot 4.x + GDScript 기반으로, 매 단계마다 실행 가능한 상태를 유지한다.

최종 목표:

```txt
3막 구조
동료 10명
주인공 카드 40장
동료 카드 80장
장비/여관/상점/이벤트/보물 시스템
PC/모바일 가로 화면 대응
```

MVP 목표:

```txt
Act 1 플레이 가능
전투 가능
카드 보상 가능
동료 3명 임시 영입 가능
여관/상점/이벤트 최소 구현
세이브/로드 최소 구현
```

MVP 개발 전 기획 기준:

```txt
09_design_upgrade_research_and_balance.md를 수치 기준으로 사용한다.
일반 전투 3~5턴, 엘리트 5~7턴, 보스 8~11턴을 1차 목표로 한다.
동료는 자동 공격 보너스가 아니라 덱/장비/이벤트 방향을 바꾸는 빌드 축이어야 한다.
동료 영입 시 서약 전술 3개 중 1개를 선택하고, 선택한 서약 전술은 런 동안 고정한다.
서약 공격은 기본 공격력의 75%, 최소 피해 3으로 시작하고 패시브/장비의 기본 공격 보너스를 적용하지 않는다.
유대 점수는 0~100으로 누적하며 30/60/100점에서 작은 보너스를 준다.
Act 1 중간 보스 후 첫 동료를 영입하고, Act 1 보스 후 두 번째 동료를 영입한다.
Act 2 중간 지점과 Act 2 보스 후에는 새 동료가 아니라 주인공/보유 동료 강화 선택을 제공한다.
4에너지/6드로우는 더 많은 카드를 전부 쓰게 하는 규칙이 아니라, 선택지를 넓히는 규칙이다. 카드 비용 분포는 2비용 이상을 충분히 포함한다.
회복 카드는 여관을 대체하지 않도록 약하거나 조건부여야 하며, Act 2부터 일부 적은 치유 감소를 사용한다.
세계관은 스토리텔링형으로 구현하지 않고, 증표 UI/짧은 이벤트 문구/변이체 적/버려진 고대 성의 아트 방향으로 전달한다.
난이도는 우선 표준 기준만 구현하고, 여행자/숙련자/원정 규율은 밸런스 지표가 쌓인 뒤 확장한다.
```
