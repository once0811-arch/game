# 01. 게임 디자인 개요

## 1. 한 줄 설명

마법 재난으로 붕괴 중인 중세 판타지 세계에서 균형형 주인공이 동료와 서약 전술을 맺고 유대 점수를 쌓으며 3막을 돌파하는, **동료 시스템 중심의 싱글플레이 2D 로그라이크 덱빌딩 게임**.

## 2. 플랫폼과 과금

```txt
플랫폼: PC + 모바일
화면 방향: 둘 다 가로 화면
과금: 게임 구매형
인앱결제: 없음
광고: 없음
멀티플레이: 없음
```

## 3. 엔진과 개발 방식

```txt
엔진: Godot 4.x
언어: GDScript
개발 방식: Codex 기반 자연어 바이브코딩
아트워크: 전문가 제작 예정
```

## 4. 세계관

```txt
장르: 중세 판타지
세부 세계관: 마법 재난으로 붕괴 중인 세계에서 버려진 고대 성의 코어를 향하는 생존 판타지 로드무비
주인공 설정: 발굴과 무관한 낡은 현실형 용병
```

현재 세계관은 시스템과 아트 방향을 지지하기 위한 기준이다. 본 게임은 스토리텔링형 게임이 아니므로, 세계관은 긴 컷신보다 배경, 적 디자인, 장비명, 이벤트 한두 줄, UI 소품으로 전달한다.

```txt
괴물 재난 속 로드무비
낡은 현실형 용병
피로 찍은 증표
계약으로 시작해 신뢰로 바뀌는 동료 관계
위험한 여관과 낡은 상점
버려진 장비와 발굴 흔적
세계 구원보다 먼저 생존이 있는 여정
```

핵심 사건과 기능은 다음과 같다. 세부 시각 방향은 `../art_director_game_design_brief.md`를 기준으로 한다.

```txt
재난 원인: 왕립 학회와 상단이 버려진 고대 성을 발굴하다 고대 마법 코어를 작동시킴.
현재 상태: 왕국 지휘 체계, 왕립 학회, 상단 모두 사실상 붕괴.
주인공 동기: 죽은 왕국 지휘관의 증표와 작전서를 발견하고 코어 파괴 방법을 알게 됨.
최종 목표: 버려진 고대 성 내부의 고대 마법 코어를 부순다.
증표: 피로 지문을 찍는 금속패 계약 토큰. 약속을 못 지키면 지문이 검게 변한다.
```

### 4.1 Act별 분위기

```txt
Act 1: 붕괴한 외곽길
- 변이된 상단원, 왕립 학자, 용병, 동식물, 버려진 초소와 거래소.
- 플레이어는 아직 혼자지만, 동료 후보들의 흔적을 발견한다.

Act 2: 뒤틀린 탐사로
- 폐광, 뒤틀린 숲길, 학회 임시 연구소 잔해, 상단 물자 창고 폐허.
- 첫 동료와의 시너지가 본격화되고, 두 번째 동료 영입 또는 기존 동료와의 유대 투자가 런의 방향을 가른다.

Act 3: 버려진 고대 성
- 인간 성처럼 보이던 공간이 내부로 갈수록 비현실적으로 변한다.
- 고대 마법 코어, 강한 이계 생물, 동료들의 서약 전술과 유대가 수렴한다.
- 두 동료와 장비 조합을 완성해 보스 패턴을 돌파한다.
```

## 5. 주인공

```txt
주인공 타입: 균형형
전투 정체성: 공격, 방어, 유틸을 모두 다루는 기본형
세계관 정체성: 발굴과 무관한 낡은 현실형 용병
고유 카드 수: 40장
```

주인공은 동료 시스템의 중심축이다. 주인공 자체가 너무 특이하면 동료 시스템이 흐려지므로, 전투 성능은 균형형으로 둔다. 세계관에서는 재난을 수습하려다 죽은 왕국 지휘관의 시체에서 증표와 작전서를 발견하고, 버려진 고대 성의 코어를 부숴야 한다는 사실을 알게 된 용병이다.

## 6. 핵심 차별점

본 게임은 기존 덱빌딩 로그라이크의 카드 선택/맵 선택 문법을 참고하되, 런 정체성은 동료 시스템으로 만든다.

```txt
카드 + 동료 + 서약 전술 + 유대 점수 + 장비 + 이벤트 + 여관
```

가장 중요한 차별점은 **동료를 고른 뒤, 그 동료의 서약 전술 3개 중 1개를 선택해 이번 런의 전투 리듬을 정한다**는 점이다. 재미의 중심은 아래 감각이어야 한다.

```txt
로완과 붉은 추격 서약을 골랐기 때문에 표식 집중 덱이 됐다.
세라와 검은매 연계 서약을 골랐기 때문에 0비용/연타 덱이 됐다.
엘드릭과 철벽 반격 서약을 골랐기 때문에 방어도 기반 반격 덱이 됐다.
함께 전투와 이벤트를 겪으며 유대 점수가 올라 작은 보너스와 대사가 열렸다.
두 번째 동료를 누구로 받을지에 따라 장비, 카드 보상, 상점 구매가 달라졌다.
```

## 7. 디자인 기둥

### 7.1 동료가 런의 방향을 바꾼다

동료는 단순 보너스가 아니다.

동료를 영입하면 다음이 동시에 바뀐다.

```txt
- 전투 중 매 턴 동료 기본 공격 추가
- 동료 고유 패시브 적용
- 동료 서약 전술 3개 중 1개 선택
- 동료별 유대 점수 0~100 누적
- 동료 전용 카드풀 해금
- 동료 장비 슬롯 3개 추가
- 동료 관련 이벤트 가능성 증가
```

동료는 반드시 두 층의 의미를 동시에 가진다.

```txt
전술적 의미: 카드 보상과 전투 행동을 바꾸는 빌드 축.
정서적 의미: 이벤트와 여정에서 기억되는 캐릭터.
```

동료가 전투에서 방치되는 느낌을 줄이기 위해, 단일 대상 공격이나 지휘 카드는 그 턴의 **전술 표식** 대상을 만든다. 동료는 턴 종료 후 전술 표식 대상을 우선 공격한다.

### 7.2 서약 전술이 동료 선택을 전투 리듬으로 바꾼다

각 동료는 3개의 서약 전술을 가진다. 영입 시 플레이어는 그중 1개를 선택하고, 선택한 서약 전술은 해당 런 동안 고정된다.

세계관상 서약 전술은 동료와 함께 증표에 맺는 이번 여정의 전투 조항이다. 제안자와 승낙자가 피로 지문을 찍는 계약 문화에서 출발하지만, 게임 규칙상 서약 전술 실패나 검은 지문 패널티를 매 전투에 적용하지는 않는다.

```txt
서약 전술은 전투마다 랜덤으로 바뀌지 않는다.
서약 전술은 업그레이드되지 않는다.
서약 전술은 동료의 성격과 덱 방향을 알려주는 고정 규칙이다.
```

예:

```txt
로완: 같은 적을 2회 이상 공격하면 추가 공격.
세라: 비용 0 카드를 2장 이상 쓰면 추가 공격과 드로우.
엘드릭: 방어도 기준을 넘기면 방어 일부를 피해로 전환.
```

### 7.3 유대 점수가 여정을 기억하게 만든다

동료는 함께 전투와 이벤트를 겪으며 유대 점수를 얻는다.

```txt
유대 점수 범위: 0~100
보너스 구간: 30 / 60 / 100
보너스 성격: 기본 공격력 +1, 패시브 소폭 강화, 대사/이벤트 가중치 같은 작은 변화
```

유대 점수는 동료와 함께했다는 감각을 주기 위한 시스템이다. 전투력을 크게 폭발시키는 성장축이 되면 안 된다.

세계관상 유대 점수는 계약이 인간적 신뢰로 바뀌는 정도다. 30/60/100 구간은 증표의 장식, 피 지문, 끈, 흠집이 정돈되는 식의 작은 시각 변화로 표현한다.

### 7.4 이벤트가 런의 변수를 만든다

런의 변동성은 이벤트, 여관, 보물/특수보상, 상점 거래가 나눠 담당한다.

```txt
일반 이벤트
이벤트 여관
보물/특수보상 방
일반 전투 후 10% 특수보상
상점 이벤트성 거래
```

### 7.5 골드는 생존 자원이다

골드는 단순 구매 재화가 아니다.

```txt
상점에서 카드/장비/강화 구매
여관에서 회복
이벤트 선택지 비용
카드 제거/복제/변화 비용
```

골드를 어떻게 쓰느냐가 생존과 덱 완성도를 동시에 결정한다.

### 7.6 장비는 동료 영입의 가치를 높인다

장비 슬롯은 주인공과 동료에게 나뉜다.

```txt
주인공 기본 장비 슬롯: 3개
동료 1명당 추가 장비 슬롯: 3개
최대 동료 2명
최대 장비 슬롯: 9개
```

따라서 Act 2에서 새 동료를 영입할지, 기존 동료와의 유대를 더 밀지의 선택은 다음 의미를 갖는다.

```txt
새 동료 영입 = 전투 인원 증가 + 카드풀 증가 + 새 서약 전술 + 장비 슬롯 증가
기존 동료 유대 투자 = 슬롯은 늘지 않지만 기존 서약/패시브 방향에 작은 보너스
```

장비는 무제한 누적 보너스가 아니라, 아래 질문을 만들기 위한 제한된 슬롯 퍼즐이다.

```txt
이 장비를 주인공에게 줄 것인가, 동료에게 줄 것인가?
팀 전체 효과가 좋은가, 특정 동료 전용 효과가 좋은가?
두 번째 동료를 받으면 장비 슬롯 3개가 늘어나는데, 지금 장비 인벤토리가 그 가치를 살릴 수 있는가?
```

### 7.7 골드가 선택을 날카롭게 만든다

골드는 항상 세 방향으로 압박해야 한다.

```txt
체력 보존: 여관
덱 품질: 카드 제거/강화/복제/변화
런 정체성: 동료 카드/장비/이벤트 거래
```

상점과 여관은 쉬어가는 화면이 아니라, 다음 3~4노드의 위험을 계산하는 전략 화면이다.

### 7.8 Act 1에서도 동료의 맛을 보여준다

정식 영입은 Act 1 보스 후에 진행하지만, 게임의 차별점이 너무 늦게 드러나면 안 된다.

따라서 Act 1 중반에는 동료 후보를 예고하는 이벤트를 1회 이상 제공한다.

```txt
동료 후보의 흔적, 전투 스타일, 장비 궁합을 보여준다.
정식 영입, 동료 카드 추가, 동료 슬롯 점유는 하지 않는다.
다음 전투 1회성 보정 정도만 제공한다.
Act 1 보스 후 후보 3택에는 예고로 본 동료가 최소 1명 포함된다.
```

## 8. 레퍼런스에서 가져오지 않는 시스템

아래 시스템은 본 게임에 없다. 문서에서는 비교나 금지 규칙을 설명할 때만 짧게 언급한다.

```txt
유물
포션
저주 카드
상태 카드
전투 중 카드 생성
선택적 4막
진엔딩 키
승천 난이도
일일 도전
커스텀 모드
무한 모드
점수 시스템
통계 화면
리더보드
모드 지원
시드 입력/공유
```

## 9. 기본 게임 루프

```txt
런 시작
→ 맵 노드 선택
→ 전투 / 이벤트 / 상점 / 여관 / 보물 / 엘리트 진행
→ 보상 선택
→ 다음 노드 선택
→ Act 보스전
→ Act 1/2 보스 후 동료 영입 또는 기존 동료 유대 보상
→ Act 3 보스 처치 시 엔딩
```

## 10. 전투 루프

```txt
전투 시작
→ 플레이어 턴 시작
→ 카드 6장 드로우
→ 에너지 4 획득
→ 카드 사용 및 전술 표식 지정
→ 선택한 서약 전술 조건 검사/발동
→ 턴 종료
→ 동료들이 전술 표식 또는 최저 체력 적을 순서대로 기본 공격
→ 적 턴
→ 다음 턴 반복
```

## 11. 승리/패배 조건

```txt
전투 승리: 모든 적 처치
런 클리어: Act 3 보스 처치
전투 패배: 주인공 체력 0
런 패배: 주인공 체력 0
```

동료는 별도 체력이 없다. 주인공과 체력을 공유한다.

## 12. MVP 목표

MVP는 아래를 목표로 한다.

```txt
1명의 주인공
동료 3명 임시 구현
Act 1만 구현
기본 전투
서약 전술 3택
카드 보상
동료 영입
여관/상점/이벤트 최소 구현
세이브/로드 최소 구현
```

정식 1차 목표는 다음이다.

```txt
3막 전체
동료 10명
주인공 카드 40장
동료 카드 80장
장비 시스템
서약 전술 시스템
유대 점수 시스템
이벤트/여관/상점/보물 시스템
보스 3종
```

## 13. 재미 검증 목표

첫 내부 플레이테스트 전 기준 목표는 다음이다.

```txt
일반 전투는 3~5턴 안에 끝난다.
```
